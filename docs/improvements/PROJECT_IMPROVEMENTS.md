# Project Improvements

## 목표
- 기능 추가보다 먼저 품질 게이트와 운영 안정성을 단계적으로 올립니다.
- 각 단계는 "실행 항목 + 완료 기준"으로 관리합니다.

## Phase 1. 품질 게이트 고도화
1. 통합 테스트 작성
- 흐름 기준: `source 등록 -> sync -> posts 조회`
2. 프론트 E2E 스모크 테스트 작성
- 최소 사용자 경로: 목록 진입 -> 필터 적용 -> 상세 진입
3. CI 품질 게이트 고정
- `backend ./gradlew test`
- `frontend npm run build`

완료 기준
- 실패를 재현하는 테스트 케이스 1개 이상 포함
- PR에서 테스트 실패 시 병합 차단 확인

## Phase 2. 운영 안정성 강화
1. 수집 실패 원인 분류 체계 정착
- `sync_jobs.error_message`를 네트워크/파싱/데이터 오류로 구분
2. 수집 성능 측정 기준 고정
- 소스별 `parsedCount`, `savedCount`, 평균 소요시간 집계
3. 검색 성능 점검
- 검색 쿼리 실행 계획 확인 및 튜닝 항목 정리

완료 기준
- 실패 사유 상위 3개를 주기적으로 집계 가능
- 주요 조회 API p95(목표 300ms) 추적 가능

## Phase 3. 운영 성숙도 강화
1. 비밀값 관리 표준화
- 관리자 기본 계정 제거, 환경별 주입 방식 문서화
2. 백업/복구 리허설
- PostgreSQL 백업/복구 1회 이상 실습 및 로그 보관
3. 배포 점검 체크리스트 고정
- 배포 전/후 확인 항목, 롤백 조건, 헬스체크 절차 표준화

완료 기준
- 복구 리허설 기록 보관
- 배포 체크리스트 문서 고정 및 운영 반영

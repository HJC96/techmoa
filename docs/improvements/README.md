# Improvements Index

개선사항을 한 번에 보지 않고 영역별로 확인할 수 있도록 분리한 문서 모음입니다.

## 문서 목록
- `docs/improvements/PROJECT_IMPROVEMENTS.md`: 전사(공통) 우선순위와 단계별 완료 기준
- `docs/improvements/BACKEND_IMPROVEMENTS.md`: API/수집/데이터 모델 중심 개선 항목
- `docs/improvements/FRONTEND_IMPROVEMENTS.md`: UI/흐름 검증 및 사용자 경험 개선 항목
- `docs/improvements/INFRA_IMPROVEMENTS.md`: 배포/보안/운영 자동화 개선 항목

## 권장 확인 순서
1. `docs/improvements/PROJECT_IMPROVEMENTS.md`
2. `docs/improvements/BACKEND_IMPROVEMENTS.md`
3. `docs/improvements/FRONTEND_IMPROVEMENTS.md`
4. `docs/improvements/INFRA_IMPROVEMENTS.md`

## 업데이트 원칙
1. 별도 요청이 없어도 작업 완료 시 관련 카테고리 문서에 즉시 반영한다.
2. 변경이 여러 영역에 걸치면 각 카테고리 문서에 나눠 기록한다.
3. 이력은 `날짜 -> 작업 단계 -> 검증 결과` 순서로 작성한다.
